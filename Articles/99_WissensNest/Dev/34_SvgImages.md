# WissensNest

## SVG Images — Design and Implementation

SVG images can be embedded in Knowledge Workbench article blocks and managed through a dedicated library browser. They are stored in the database, referenced by GUID in block Markdown, and rendered inline by a custom Markdig pre-processor.

---

### Why SVGs, not Files

- Article blocks are pure Markdown stored in SQLite. Embedding raw SVG XML directly would pollute content search and break plain-text rendering.
- A separate `SvgImages` table stores the SVG content by GUID. Block content holds only a lightweight reference: `![alt text](svgimg://guid)`.
- The image is fetched from the DB at render time, not stored in the block — so a library-level in-place update propagates to all references immediately.

---

### Reference Scheme

SVGs are referenced in block `Content` using standard Markdown image syntax with a custom URI scheme:

```markdown
![Flow diagram](svgimg://3fa85f64-5717-4562-b3fc-2c963f66afa6)
```

`MarkdownContent.razor` pre-processes this pattern before passing content to Markdig:

1. Regex scans for `![alt](svgimg://guid)` matches.
2. Looks up each GUID in the `SvgContents` dictionary (loaded by `ArticleEditor` on article open).
3. Replaces with `<figure class="inline-svg" data-svg-id="guid">RAW_SVG</figure>`.
4. Strips `<?xml?>` and `<!DOCTYPE>` declarations for safe inline embedding.
5. Falls back to `<em class="svg-missing">[SVG not loaded]</em>` if the GUID is not in the cache.

Markdig passes raw HTML blocks through unchanged, so the `<figure>` element renders correctly.

---

### Data Model

```text
SvgImages
  Id             GUID PK
  Name           TEXT nullable
  Description    TEXT nullable
  SvgContent     TEXT NOT NULL        — raw SVG XML
  SourceBlockId  GUID nullable FK     — block the SVG was first created from (SetNull on block delete)
  ThumbnailPng   BLOB nullable        — 128 px PNG generated by Svg.Skia; null for imports before Stage D
  CreatedAt / UpdatedAt / IsDeleted
```

No join table — one SVG image can appear in many blocks (by GUID reference), but modifying a library image is global (all blocks using it see the change). The block editor's Replace action uses copy-on-edit to preserve old references.

---

### Thumbnail Generation

`SvgThumbnailHelper` (static, `WissensNest.API/Services/`) uses `Svg.Skia` 5.0.0 + SkiaSharp 3.x:

```csharp
using var svg = new SKSvg();
var picture = svg.FromSvg(svgContent);    // SVG string → SKPicture
// compute scale to fit 128 px square
// render to SKSurface → SKImage → PNG bytes
```

Generated automatically on:

- `POST /svg-images` (single create)
- `POST /svg-images/bulk-import` (ZIP import)
- `PATCH /svg-images/{id}/content` (in-place replace)

Returns `null` on any failure — callers treat a missing thumbnail gracefully (fall back to inline SVG).

### Stage A — DB + API

Migrations: `20260605125331_AddSvgImages`, `20260605160916_AddSvgThumbnails`.

API surface:

| Method | Route | Notes |
| --- | --- | --- |
| GET | `/svg-images` | All images with thumbnails |
| GET | `/svg-images/{id}` | Single image |
| GET | `/svg-images/search?q=&limit=` | Name, description, + block-context search |
| POST | `/svg-images` | Create; thumbnail auto-generated |
| POST | `/svg-images/bulk-import` | Multipart ZIP; each `.svg` → record |
| PATCH | `/svg-images/{id}` | Update name/description only |
| PATCH | `/svg-images/{id}/content` | In-place SVG content update; thumbnail regenerated |
| DELETE | `/svg-images/{id}` | Soft-delete; 409 if referenced |
| GET | `/articles/{id}/export-svgs` | ZIP of all SVGs referenced in article |

The `SearchAsync` repository method performs two queries and merges results:

1. SQL-side: records where `Name` or `Description` contains the query.
2. In-memory: blocks whose `Content` contains the query; extract all `svgimg://guid` refs; load those images.

`IsReferencedAsync` scans `Block.Content` for `svgimg://id` via `AnyAsync` — one fast SQLite call.

### Stage B — Block Editor

**SVG cache** is loaded in `ArticleEditor.LoadArticleAsync()`:

- Regex scans all block content for `svgimg://guid` patterns.
- `GetSvgImageAsync(id)` called once per unique GUID (sequential; articles typically reference < 10 SVGs).
- Two dictionaries: `_svgContentCache` (Guid → SvgContent, for Markdig) and `_svgNameCache` (Guid → Name, for the ref panel).

**`SvgPickerModal`** tabs:

- **Paste XML** — raw textarea with live SVG preview; validates `<svg` presence on confirm.
- **Upload File** — Blazor `InputFile`; reads as text (`StreamReader`); auto-fills name from filename.
- **Library** — debounced search (`Task.Delay(300)`); thumbnail grid; click to select; no API call on confirm (returns existing `SvgImageInfo`).

**Insert flow**: `+ SVG` button in edit-mode actions → picker opens → on confirm → `CreateSvgImageAsync` (for Paste/Upload) or direct use of existing record (for Library) → insert `![name](svgimg://guid)` at `getTextareaCursorPosition` → restore cursor via `_pendingCursorPosition`.

**Replace flow** (from referenced SVGs panel): opens picker in copy-on-edit mode → new `SvgImage` created → `_blockDraft` regex-replaced to swap old GUID for new GUID. Original record and any other blocks referencing it are untouched.

---

### Stage C — SVG Library Page (`/svg-library`)

Navigation: **⬡ Images** tab in `SidePanel.razor` — calls `Nav.NavigateTo("/svg-library")`, active when URL contains `/svg-library`.

**In-place replace** (library page) uses `SvgPickerModal` with `UpdateTargetId` set. The modal's `ConfirmAsync` calls `UpdateSvgImageContentAsync` instead of `CreateSvgImageAsync`. The thumbnail is regenerated on the server.

**Where used** loads lazily on first expand:

```csharp
var refs = await AiClient.SearchReferencesAsync($"svgimg://{id}", 20);
var blockRefs = refs.Where(r => r.Kind == "block").ToList();
```

Results link directly to the block anchor (`/article/{articleId}#block-{blockId}`).

---

### Stage D — Bulk Operations

**Bulk ZIP import** (`POST /svg-images/bulk-import`):

- Server extracts `ZipArchive` entries with `.svg` extension.
- Each entry: reads content, creates `SvgImage`, generates thumbnail.
- Returns the full `SvgImageInfo` list (including thumbnails) in one response.
- UI: `<label>` wrapping hidden `<InputFile>` → `HandleZipImport` → `BulkImportSvgImagesAsync(stream)` (multipart POST).

**Article SVG export** (`GET /articles/{id}/export-svgs`):

- Scans block content with `_svgRefRegex` to collect referenced GUIDs.
- Loads each image from the SVG repo.
- Packs into a `ZipArchive` in memory (collision-safe filename handling).
- Returns `application/zip` response or 204 if no SVGs referenced.
- UI: **Export SVGs** button in ArticleEditor header, visible when `_svgContentCache.Count > 0`.
